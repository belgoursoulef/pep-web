POLITIQUE DE CONFIDENTIALITÉ

Dernière mise à jour : 18 Juin 2026

1. Contexte du projet

Ce site Web a été développé dans un but pédagogique dans le cadre des projets **SAE 2.1 & 2.4 (Projet Pépinière)**. Les données collectées et traitées le sont uniquement à des fins de simulation et d'évaluation académique.

2. Données collectées

Nous collectons et traitons deux types de données :

  • Journaux de contrôle d'accès : L'identifiant du badge des employés scannés (ex: EMP001), le nom complet, le département, le statut d'accès (Autorisé/Refusé), et l'heure du passage.
  • Données de session technique : Langue préférée (cookie de session) et statut de connexion de l'administrateur.

3. Utilisation des données

Les journaux d'accès physiques et les logs d'intrusion de mouvement sont uniquement insérés dans la base de données MySQL locale et affichés sur le Tableau de bord de surveillance en direct. Aucune donnée n'est partagée avec des tiers ou des partenaires commerciaux.

4. Droits des utilisateurs

Conformément aux directives du RGPD appliquées aux projets universitaires, tout étudiant ou membre du personnel dont le badge est utilisé dans la base de données de test dispose d'un droit d'accès et de suppression de ses données sur simple demande auprès de l'équipe projet.

PRIVACY POLICY

Last updated: June 18, 2026

1. Project Context

This website was built for educational purposes as part of the **SAE 2.1 & 2.4 Pépinière project** curriculum. All collected and processed data serves purely simulated and academic evaluation goals.

2. Data We Collect

We collect and process two categories of data:

  • Access Control Logs: Scanned employee badge ID (e.g., EMP001), full name, department, authorization status (Granted/Denied), and access timestamps.
  • Technical Session Data: User language preference (stored in session cookies) and administrator login status.

3. Data Usage

Access logs and movement intrusion logs are stored locally in the MySQL database and displayed inside the live Security & Monitoring panel. No data is shared with third parties or external commercial services.

4. User Rights

In compliance with GDPR guidelines applied to academic settings, any student or staff member whose badge ID is represented in the test database has the right to access and request deletion of their records by contacting the development team.
